## 프로젝트 개요

**Enterprise AI Hub**는 대기업 업무 자동화를 목표로 한 AI 기반 사내 포털 시스템입니다.  
각 부서(HR·재무·CS·법무·R&D)의 반복 업무를 OpenAI API(gpt-4o-mini)와 연동하여 자동화합니다.

**Tech Stack**
- Frontend: Vite + React (Functional) + TailwindCSS + Recharts
- Backend: FastAPI (Python) + pg8000 (PostgreSQL)
- AI: OpenAI API (gpt-4o-mini)
- 인증: localStorage 세션 기반 (사번+부서+직급)

## Phase 2. 재무/회계팀 (Finance)

### 지시 #004 — DB 스키마 설계
```
재무팀 테이블을 설계해줘.
영수증 전표(finance_transactions), 예산(finance_budgets), 감사 로그(finance_audit_logs)가 필요해.
total_amount는 amount + tax_amount로 자동 계산되게 해줘.
```

**결과물** (`backend/tables/finance/finance_schema.sql`)
- `finance_transactions` — `total_amount GENERATED ALWAYS AS (amount + tax_amount) STORED`
- `finance_budgets` — `UNIQUE (fiscal_year, department, account_code)`
- `finance_audit_logs` — `risk_level CHECK IN ('safe','warning','danger')`
- 인덱스 3개 + `updated_at` 자동 갱신 트리거

### 지시 #005 — 영수증 OCR 자동화 (AccountantPage)
```
영수증 이미지를 올리면 AI가 자동으로 항목, 금액, 부가세, 계정과목을 추출해줘.
중복 영수증 탐지도 해줘. 분석 결과를 사람이 검토하고 확정하는 Human-in-the-Loop 구조로 만들어줘.
```

**결과물**
- `POST /api/finance/ocr` — 이미지 저장 + GPT Vision 분석 + 중복 탐지
- `POST /api/finance/transactions` — 분석 결과 DB 저장
- `PUT /api/finance/transactions/{id}` — 계정과목·금액 수정
- `POST /api/finance/transactions/{id}/confirm` — 전표 확정 (`status → confirmed`)
- `GET /api/finance/transactions/export` — 확정 전표 엑셀 다운로드 (openpyxl)
- `AccountantPage.jsx` — 드래그&드롭 업로드 + 미리보기 + 수정 + 확정 UI

### 지시 #006 — AI 계정과목 추천

### 지시 #007 — 재무/자금 대시보드 (TreasuryPage) ← 최신
```
TreasuryPage를 구현할 건데, 재무부 소속 직원만 들어올 수 있게 해줘.
Accountant 페이지에서 등록된 지출 데이터를 기반으로 부서별 통계를 내서 차트로 보여줘.

1. 접근 권한: department='재무팀' or position='대표이사'만 접근 가능 (프론트 가드 + 서버 검증)
2. 백엔드 통계 API: 부서별 지출 합계 + 예산 대비 집행률, 월별 추이, 감사 위험 로그

4. 차트: Recharts Pie Chart(지출 비중) + Area Chart(월별 추이)
5. AI CFO 리포트: gpt-4o-mini로 예산 초과 위험 부서 분석 및 조언 생성
```

**결과물**
- `GET /api/finance/stats` — `x-employee-id` 헤더로 DB 부서 검증, 5개 통계 집계

- `POST /api/finance/seed` — 부서별 샘플 30건 + 예산 20건 삽입
- `TreasuryPage.jsx` — Pie/Area 차트 + 예산 소진 바 + AI 리포트 패널

## 바이브코딩 패턴 분석

이 프로젝트에서 반복적으로 사용한 AI 지시 패턴입니다.

### 패턴 1 — "역할 부여 후 요구사항 도출"
```
대기업의 [부서명] 실무자라고 생각하고, AIFDE에게 어떤 업무 자동화를 부탁할 것 같아?
```
→ AI가 실무 관점으로 기능 목록을 제안하게 한 뒤, 선별해서 구현 지시

### 패턴 2 — "스키마 기준 명시"
```
데이터는 첨부된 finance_schema.sql에 정의된 컬럼명을 정확히 사용해줘.
(total_amount, receipt_date, account_code 등)
```
→ DB 스키마를 먼저 확정하고 그것을 기준으로 프론트·백엔드를 동시에 지시

### 패턴 3 — "번호 매긴 요구사항"
```
1. 접근 권한: ...
2. 백엔드 API: ...
3. 테스트 데이터: ...
4. 프론트엔드: ...
5. AI 연동: ...
```
→ 한 번의 지시에 5개 레이어를 동시에 요청해서 컨텍스트 일관성 유지

### 패턴 4 — "Human-in-the-Loop 명시"
```
AI 분석 결과를 사람이 검토하고 확정하는 구조로 만들어줘.
```
→ AI 자동화 + 사람 검증 단계를 항상 포함시켜 실무 신뢰성 확보

### 패턴 5 — "권한 계층 명시"
```
재무팀 소속 직원만 들어올 수 있게 해줘.
프론트 라우터 가드 + 서버 사이드 검증 둘 다 해줘.
```
→ 클라이언트·서버 이중 보안을 명시적으로 요청

## 핵심 설계 원칙

1. **API 중앙화** — 모든 fetch는 `frontend/src/api/*.js`에서만 호출
2. **Loading/Error/Empty 상태** — 모든 비동기 UI에 3가지 상태 필수 구현
3. **Mobile-First** — 버튼 최소 44px, 반응형 grid
4. **보안** — `.env`로 시크릿 관리, 하드코딩 금지
5. **한글 주석** — 모든 코드 주석은 한국어
6. **Human-in-the-Loop** — AI 분석 후 사람이 검토·확정하는 단계 포함
