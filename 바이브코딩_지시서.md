## 포트폴리오 개요

**목표 직군**: FDE (AI Frontend Developer Engineer)  
**포트폴리오 유형**: 과정중심형 — 기획·지시·구현·회고의 전 과정을 기록

**Enterprise AI Hub**는 대기업 업무 자동화를 목표로 한 AI 기반 사내 포털 시스템입니다.  
각 부서(재무·HR·CS·법무·영업·구매·전략·R&D)의 반복 업무를 OpenAI API(gpt-4o-mini)와 연동하여 자동화합니다.

**Tech Stack**
- Frontend: Vite + React (Functional) + TailwindCSS + Recharts
- Backend: FastAPI (Python) + pg8000 (PostgreSQL)
- AI: OpenAI API (gpt-4o-mini, text-embedding-3-small, GPT Vision)
- 인증: localStorage 세션 기반 (사번+부서+직급)

### 지시 #004 — DB 스키마 설계
```
재무팀 테이블을 설계해줘.
영수증 전표(finance_transactions), 예산(finance_budgets), 감사 로그(finance_audit_logs)가 필요해.

```

**결과물** (`backend/tables/finance/finance_schema.sql`)
- `finance_transactions` — `total_amount GENERATED ALWAYS AS (amount + tax_amount) STORED`
- `finance_budgets` — `UNIQUE (fiscal_year, department, account_code)`
- `finance_audit_logs` — `risk_level CHECK IN ('safe','warning','danger')`
- 인덱스 3개 + `updated_at` 자동 갱신 트리거

### 버그픽스 #032-a — 툴 카드 중복 표시 버그
```
예산잔액조회랑 상품 가격 판매처 검색이 가끔씩 2개씩 뜨는 버그가 있어.

```

**원인**: GPT가 같은 툴을 여러 라운드에서 반복 호출할 때 프론트엔드가 새 카드를 계속 추가함

**수정 내용**
- `onToolStart`: 같은 `tool` 이름의 카드가 이미 존재하면 새로 추가하지 않고 기존 카드를 `running`으로 덮어씀
- `onToolDone`: 툴 이름으로만 매칭해서 해당 카드를 `done`으로 업데이트
- 결과: GPT가 동일 툴을 몇 번 호출하든 카드는 **툴 종류당 1개**만 유지

```
데이터는 첨부된 finance_schema.sql에 정의된 컬럼명을 정확히 사용해줘.
(total_amount, receipt_date, account_code 등)
```
→ DB 스키마를 먼저 확정하고 그것을 기준으로 프론트·백엔드를 동시에 지시

### 패턴 3 — "번호 매긴 요구사항"
```
1. 접근 권한: ...
2. 백엔드 API: ...
3. 테스트 데이터: ...
4. 프론트엔드: ...
5. AI 연동: ...
```
→ 한 번의 지시에 5개 레이어를 동시에 요청해서 컨텍스트 일관성 유지

### 패턴 4 — "Human-in-the-Loop 명시"
```
AI 분석 결과를 사람이 검토하고 확정하는 구조로 만들어줘.
```
→ AI 자동화 + 사람 검증 단계를 항상 포함시켜 실무 신뢰성 확보

### 패턴 5 — "권한 계층 명시"
```
재무팀 소속 직원만 들어올 수 있게 해줘.
프론트 라우터 가드 + 서버 사이드 검증 둘 다 해줘.
```
→ 클라이언트·서버 이중 보안을 명시적으로 요청

### 패턴 6 — "공통 모듈 재활용 지시"
```
기존 [A 서비스]와 구조가 거의 동일하니까 공통 모듈로 분리하고, [B 전용 서비스]를 새로 만들어줘.
```
→ 코드 중복을 직접 인지하고 리팩터링과 신규 구현을 동시에 지시

### 패턴 7 — "스트리밍 + 파일 다운로드 조합"
```
실시간 스트리밍으로 진행 상황을 보여주고, 완료 후 [PPTX/DOCX/PDF]로 다운로드할 수 있게 해줘.
```
→ UX 피드백(스트리밍)과 결과물 산출(파일 다운로드)을 하나의 지시에 묶어서 요청

### 패턴 8 — "불편한 UX를 구체적 동작으로 재정의"
```
하나씩 검색해서 추가하는 방법은 어때? 티커 탐색은 기존이랑 똑같이 하고.
```
→ "불편하다"는 피드백에 대안 동작을 직접 제안 → AI가 입력 플로우를 재설계

### 패턴 9 — "액션 트리거 기반 AI 자동 실행"
```
회사를 하나 추가하면 AI가 자동으로 경쟁사를 추천해줘.
```
→ 사용자 액션(추가)을 트리거로 AI 기능이 자동 실행되는 구조 요청 — 별도 버튼 없이 컨텍스트 인지형 UX 구현

## 핵심 설계 원칙

1. **API 중앙화** — 모든 fetch는 `frontend/src/api/*.js`에서만 호출
2. **Loading/Error/Empty 상태** — 모든 비동기 UI에 3가지 상태 필수 구현
3. **Mobile-First** — 버튼 최소 44px, 반응형 grid
4. **보안** — `.env`로 시크릿 관리, 하드코딩 금지
5. **한글 주석** — 모든 코드 주석은 한국어
6. **Human-in-the-Loop** — AI 분석 후 사람이 검토·확정하는 단계 포함
7. **공통 모듈 우선** — document_parser, rag_utils, stt_service 등 재사용 구조
8. **실시간 피드백** — 장시간 AI 작업은 SSE 스트리밍으로 진행 상황 표시
